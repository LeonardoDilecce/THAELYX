# Security Policy

## Supported Versions
Security updates and fixes will be applied to the latest active version of **T.H.A.E.L.Y.X**  
Older versions may not receive patches unless the issue is critical.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please follow these guidelines:

1. **Do not open a public issue.**  
   Security reports must be handled privately to protect users.

2. **Contact the maintainers directly:**  
   Please send a detailed report to:  
   ****

3. Include as much information as possible:
   - Description of the vulnerability  
   - Steps to reproduce  
   - Potential impact  
   - Suggested fixes (if any)

We will acknowledge your report within **3 Months** and provide an estimated timeline for investigation and resolution.

---

## Responsible Disclosure

We ask that you:
- Do not publicly disclose the vulnerability before it is fixed  
- Do not exploit the vulnerability  
- Do not test on systems or data you do not own  

Following responsible disclosure helps keep the project and its users safe.

---

## Security Fix Process

Once a vulnerability is confirmed:
1. The maintainers will create a private patch.  
2. A fix will be tested and validated.  
3. A new release will be published.  
4. A note will be added to the `CHANGELOG.md` describing the fix (without sensitive details).  

---

## Thank You

We appreciate your help in keeping **T.H.A.E.L.Y.X** secure and reliable.  
Your contributions make the project stronger for everyone.
